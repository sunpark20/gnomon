# PRD — Gnomon

> 작성: 2026-04-20 · 상태: **v0.4 (Auto 통일 + 멀티 v2 + 키보드 인터랙션)**
>
> 관련 문서:
> - [BACKGROUND.md](BACKGROUND.md) — 제품 스토리 (홈페이지/마케팅용)
> - [research/adaptive-curves.md](research/adaptive-curves.md) — 곡선 학술 근거 5,000단어

## 1. 제품 정의

한 문장: **"맥북 내장 조도센서로 외장 모니터의 밝기와 대비를 자동 조정하고, 수동 제어도 가능한 macOS 네이티브 앱."**

**이름의 의미**: Gnomon (노-몬) = 해시계에서 그림자를 드리우는 삼각형 바늘. "빛의 세기를 읽어 화면을 맞추는 작은 부품"이라는 의미. 모니터 스탠드의 수직성과도 시각적으로 맞는 메타포.

Lunar의 Sensor Mode를 개인 사용 최소 스코프로 재구현. UI/UX는 **Lunar의 것을 상당 부분 벤치마크** (세부 설정 팝오버 포함).

## 2. 목표 / 비-목표

### 목표 (In scope)
- 조도 기반 **자동 밝기 + 대비 조정** (Adaptive 모드)
- 수동 밝기/대비 조정 (Adaptive OFF 시)
- 실시간 상태 가시화 (센서값·타겟값·실제값·카운트다운)
- **밝기/대비 단축키** 매핑
- **메뉴바 아이콘 + 클릭 시 창 토글** 패턴
- **조도/밝기 간단 로깅** (튜닝 참고용)
- 초기 실행 시 **온보딩** (Lunar 스타일 — 상태 체크 → 권한/의존성 확인)
- 내 하드웨어에서 확실히 동작: MacBook Pro (M5 Pro) + LG HDR 4K (DDC)

### 비-목표 (Out of scope, MVP)
- **색온도 자동 조정** — Night Shift / f.lux 사용 권장 (5.2.3 참조)
- 다중 외장 모니터 (LG HDR 4K 하나만)
- 앱별 프리셋
- 날씨/위치/시간대 연동
- AI Learning (기록은 하되, 학습은 하지 않음)
- Sidecar, DisplayLink, 프로젝터
- 멀티 윈도우 / 팝오버 / 탭 — **단일 화면**으로 통일 (5.6)
- Gamma/LED Color Gain/Volume OSD/FaceLight 등 Lunar 고급 기능
- 코드사인/공증/자동업데이트 (개인용이므로)

> **원칙**: 비-목표 항목은 MVP 사용 후 *실제 불편함* 생긴 것만 편입. 가정으로 미리 안 만듦.

## 3. 사용자

**유일한 사용자 = 본인.** MacBook Pro M5 Pro + LG HDR 4K 환경.

## 4. 핵심 사용자 스토리

1. **아침**: 창가 햇빛이 세지면 모니터가 알아서 밝아진다.
2. **저녁**: 해가 지면 모니터가 알아서 어두워진다.
3. **영화 모드**: Adaptive 토글 OFF → 수동 슬라이더/단축키로 빠르게 어둡게.
4. **확인**: 메뉴바 아이콘 클릭 → 창 열림 → 현재 조도, 계산된 타겟, 모니터 현재값, 다음 전송까지 남은 초를 실시간으로 확인.
5. **빠른 조정**: 창을 안 열어도 `⌃⌥⌘+↑/↓`로 밝기 즉시 조정.

## 5. 기능 요구사항

### 5.1 조도 측정
- **v1**: `Process` + `/usr/libexec/corebrightnessdiag status-info` → plist 파싱 → `AggregatedLux`
- 주기: 1초 (실시간 UI용). DDC 전송 주기와 분리.
- **v2 (추후)**: `IOHIDManager`로 `AppleSPUVD6286` HID 이벤트 구독 (반응성 향상)

### 5.2 Adaptive 곡선 (수학 검증 후 확정)

> 검증 근거: [`research/adaptive-curves.md`](research/adaptive-curves.md) §6 + 본 프로젝트 자체 수학 재검증
> (램버시안 종이 휘도 식 + LG HDR 4K 350 nits 선형 가정 + 사용자 "종이보다 살짝 밝게" 휴리스틱 정량화)

#### 5.2.1 밝기 곡선 (확정)

```
b(lux) = b_min + (b_max - b_min) × clamp(log10(lux + 1) / log10(2001), 0, 1)
```

| 파라미터 | 값 | 근거 |
|---|---|---|
| `b_min` | **20** | 야간 사용 시 너무 어두운 화면 방지 (사용자 선호) |
| `b_max` | **95** | 풀 가동 직전. 창가 직사광 환경 대응 (사용자 선호, aggressive) |
| `lux ceiling` | **2000** | 창가 직사광 대응. 1000 lux에서 조기 포화 방지 |

> 사용자 결정 (v0.4): 안전한 15/75 보다 약간 aggressive한 20/95 선호. 야간 너무 어둡지 않고, 한낮 풀가동 가능.

**샘플 결과** (LG HDR 4K, 350 nits 가정, 종이 ρ=0.85):

| 환경 | lux | b% | Monitor nits | Paper nits | 배율 |
|---|---|---|---|---|---|
| 야간 (불 끔) | 5 | 38 | 164 | 1.4 | — |
| 어두운 실내 | 50 | 59 | 227 | 13.5 | 16.8× |
| 일반 실내 | 200 | 72 | 266 | 54 | 4.9× |
| 사무실 표준 | 500 | **81** | **293** | 135 | 2.17× |
| 밝은 실내 | 1000 | 88 | 314 | 271 | 1.16× |
| 창가 | 2000 | 95 | 335 | 541 | 0.62× |

> 수정 이력: v0.4의 lux=5 행 (24 / 122)은 계산 오류. v0.5에서 공식 재계산 결과 38 / 164로 정정. 다른 행은 이상 없음.

→ "사무실 환경(500 lux)에서 모니터가 종이의 약 2.2배" — 사용자 선호 aggressive 톤 정합.

> 슬라이더로 b_min, b_max 실시간 조정 가능. 위 수치는 **공장 디폴트**일 뿐.

#### 5.2.2 대비 (확정 — 고정)

```
contrast = 70  (LG HDR 4K 출하 기본값)
```

- **자동 조정 안 함.** 학술 근거 약하고, 체감 효과 미미.
- 사용자는 슬라이더로 수동 조정 가능 (50~80 권장 범위).
- 1주일 실사용 후 정말 불편하면 v2에서 **밝기 종속 곡선 [50, 70]** 도입 평가.

#### 5.2.3 색온도 (스코프 제외 — 다른 앱 사용 권장)

**Gnomon은 색온도를 건드리지 않는다.**

이유:
1. 단일 lux 센서로 색온도 추정 원리적으로 불가
2. macOS Night Shift / f.lux가 시간 기반 색온도 조정을 이미 잘 함
3. 바퀴를 다시 발명할 이유 없음

**앱 내 안내 (온보딩)**: "색온도 자동 조정은 [f.lux](https://justgetflux.com/) 또는 macOS Night Shift를 함께 사용하세요."

### 5.3 DDC 전송
- 도구: `/opt/homebrew/bin/m1ddc` (경로 하드코딩 → 설정에서 교체 가능)
- 대상: LG HDR 4K (자동 감지 목록에서 선택)
- 주기: 사용자 설정 인터벌 (**기본 30초**, 15~120초 조정 가능)
- 조건: `abs(target - last_sent) ≥ 2`일 때만 전송 (DDC 마모 방지)
- 전송 전 처리:
  - lux EMA 스무딩 `α=0.2` (1초 샘플 → 약 5초 반응 지연)
  - 옵션: median-of-5 이상치 제거 (플리커 발생 시만)
- 옵션: 타겟 변경 시 5초 linear interpolation으로 부드러운 전이
- **대비는 자동 전송 안 함** (5.2.2). 수동 조정 시에만 DDC 전송.

### 5.4 수동 모드 / Auto-OFF 정책 (확정)

**정책 A 채택**: Auto 모드에서 사용자가 수동 조작하면 **Auto가 자동 OFF**.

상세:
- 슬라이더 드래그 → 즉시 DDC 전송 (200ms 디바운스) + Auto 토글 OFF + 펄스 애니메이션
- 단축키 (⌃⌥⌘+↑/↓) → 동일하게 Auto OFF
- 메인 UI 큰 숫자 클릭 → 인라인 텍스트 입력 모드 → 엔터 시 적용 + Auto OFF
- Auto 재개: 토글 클릭 또는 ⌃⌥⌘+A
- Auto OFF 중에는 작은 메시지 "Auto paused — tap toggle to resume"

이유: Apple 컨벤션과 일치, 가장 예측 가능, "내가 한 게 무시됐다" 혼란 제거.

> **명명 변경**: PRD 전반의 "Adaptive" → "Auto" 로 변경. UI 라벨도 "Auto"로 통일. (Apple 컨벤션 일관성)

### 5.5 단축키 (MVP 포함)
| 단축키 | 동작 |
|---|---|
| `⌃⌥⌘ + ↑` | 밝기 +5 |
| `⌃⌥⌘ + ↓` | 밝기 -5 |
| `⌃⌥⌘ + →` | 대비 +5 (수동 조정만) |
| `⌃⌥⌘ + ←` | 대비 -5 (수동 조정만) |
| `⌃⌥⌘ + A` | Adaptive 토글 |
| `⌃⌥⌘ + ,` | 창 토글 |

- 구현: `NSEvent.addGlobalMonitorForEvents` 또는 `MASShortcut` 라이브러리
- 권한: Accessibility 접근 허용 필요 (온보딩에서 안내)
- 단축키는 설정 화면에서 재할당 가능 (v2)

### 5.5.1 위트 라벨 (Lux 구간별 멘트)

조도 패널 아래 작은 글씨로 표시. 매번 무작위로 3개 중 하나.

| Lux 구간 | 분위기 | 멘트 풀 (3개씩) |
|---|---|---|
| 0-10 | Pitch dark | "어둡네요... 영혼까지 어두워지지는 마세요." / "은은한 어둠이군요. 잠은 자고 계신가요?" / "동굴에서 코딩 중이신가요?" |
| 10-50 | Very dim | "분위기 있는 조명입니다. 와인 한 잔 어때요?" / "잔잔한 무드네요. 졸음 조심하세요." / "촛불 켠 듯한 분위기입니다." |
| 50-200 | Dim indoor | "아늑한 작업실 같은 조명입니다." / "조명 하나만 더 켜셔도 좋을 것 같아요." / "독서하기 딱 좋은 빛이네요." |
| 200-500 | Office | "전형적인 사무실 조명입니다." / "야근의 향기가 살짝 나는데요?" / "표준적인 인간 활동 영역입니다." |
| 500-1000 | Bright | "밝고 활기찬 공간이네요!" / "오늘 뭔가 좋은 일 있으세요?" / "비타민 D가 슬슬 합성될 것 같습니다." |
| 1000-2000 | Soft daylight | "햇살이 따뜻하게 들어오네요." / "광합성 시작합니다 🌱" / "커튼을 살짝 닫아주셔도 좋을 듯." |
| 2000+ | Direct sunlight | "날씨가 너~~~무 좋네요. 나가서 팬티만 입고 가볍게 러닝 어때요?" / "선글라스 챙기세요. 진심이에요." / "모니터가 한계에 다다랐습니다... 살려주세요." |

구현 노트: 멘트 변경은 lux 구간이 *바뀔 때마다* 새로 추첨. 같은 구간 내에서 lux가 미세 변동해도 멘트 안 바꿈 (산만함 방지).

### 5.6 창 구조 — 단일 화면 (확정)

**모든 컨트롤을 한 창에서.** 팝오버/탭/멀티 윈도우 없음. Lunar 메인 화면처럼 정보 밀도는 적당히 낮게, 시각적 흐름은 한눈에.

**한 창에 들어갈 요소** (MVP):
1. **조도센서 패널** (좌측)
   - 현재 lux 큰 숫자 (예: `428 lx`)
   - 가로 게이지 바
   - 그 아래 위트 멘트 한 줄 (5.5.1)
2. **밝기 컨트롤** (우측 상단)
   - 큰 숫자 1개: **CURRENT** (모니터 실제값) — **클릭 시 키보드 입력 가능**
   - 작은 숫자: **TARGET** (조도 기반 계산값, Auto 모드일 때만)
   - 슬라이더 1개
   - `Auto` 토글 (한 단어로)
3. **대비 컨트롤** (우측 하단)
   - 큰 숫자 1개: 현재 대비 — **클릭 시 키보드 입력 가능**
   - 슬라이더 1개 (Min/Max 양 끝 라벨)
   - **Auto 토글 없음** — 수동 전용
4. **하단 상태 바**
   - 좌: ⏸ Pause / ⚡ Apply Now 버튼
   - 우: "Next sync in 24s ⏱" 카운트다운

**Min/Max는 메인에 표시 안 함.** 우측 상단 ⚙️ 설정 패널에서만 조정 (기본 20/95).

**키보드 인터랙션** (전 컨트롤):
- 숫자 클릭 → 인라인 입력 활성화 → 엔터/포커스 아웃 시 적용
- 슬라이더에 포커스 → ←/→ 방향키로 1씩, Shift+←/→ 로 5씩 조정
- 모든 수동 조작은 Auto OFF 트리거 (5.4)

**UI는 100ms 단위로 재렌더링** → 조도·타겟값 실시간 애니메이션.

**프로토타입**: Google Stitch에서 시안 제작 → 검토 후 SwiftUI 구현 → `ui-prototype.md`에 결과 캡처.

### 5.9 온보딩 (초기 실행)
Lunar 스타일 체크리스트:
1. `corebrightnessdiag` 실행 확인 → lux 값 잡히는지 테스트
2. `m1ddc` 설치 확인 → LG HDR 4K 감지 테스트
3. Accessibility 권한 요청 (단축키용)
4. 초기 Min/Max 설정 (기본값 **15/75** — 5.2.1 근거)
5. Lunar 동시 실행 경고 (있으면 끄라는 안내)
6. **색온도 안내**: "Gnomon은 밝기/대비만 자동화합니다. 색온도는 macOS Night Shift 또는 [f.lux](https://justgetflux.com/) 사용을 권장."

실패 시 해결 방법 안내 (Homebrew 설치 링크 등).

### 5.9.1 멀티 모니터 정책

**MVP**: 단일 모니터(LG HDR 4K) 전용. 외장 N≥2 감지 시 첫 번째만 조정 + 작은 배너로 안내.

**v2 (추후 과제)**: 사용자 통찰 — *"모니터마다 토글로 켜고 끄면 되잖아"*

같은 lux 곡선을 모니터별로 독립 적용하는 구조. 구현 윤곽:

```
struct MonitorConfig {
    let id: String          // m1ddc UUID
    let name: String        // "LG HDR 4K"
    var autoEnabled: Bool   // 모니터별 Auto on/off
    var bMin: Int           // 모니터별 Min/Max (다른 모델 대응)
    var bMax: Int
    var contrast: Int       // 수동 고정값
}

// 메인 루프
foreach monitor in detectedMonitors:
    if monitor.autoEnabled:
        target = curve(currentLux, monitor.bMin, monitor.bMax)
        m1ddc.send(monitor.id, brightness=target)
```

UI는 메인 화면에 모니터 카드를 N개 나열 (가로 스크롤 또는 세로 스택). 각 카드마다 Auto 토글.

**MVP에 포함하지 않는 이유**:
- 본인 환경은 단일 모니터
- v1 검증/안정화 우선
- 멀티 모니터 시 활성 디스플레이 감지·DDC 동시 전송 안정성 등 별도 검증 필요

### 5.9.2 다른 디스플레이 앱과의 충돌

**기본 macOS 동작**: F1/F2 키 = 내장 디스플레이 밝기. 외장 영향 없음 → Gnomon과 무충돌.

**충돌 가능 시나리오**:
- Lunar/MonitorControl이 F1/F2를 외장에 매핑한 경우 → 동시 동작 시 마지막 명령이 이김 (DDC race)
- 온보딩 5단계에 안내: "다른 디스플레이 앱이 켜져 있다면 종료하거나 외장 매핑을 해제하세요"

### 5.10 로깅 (간단 기록만, 학습 없음)
- 위치: `~/Library/Application Support/Gnomon/log.csv`
- 포맷: `timestamp, raw_lux, ema_lux, target_brightness, sent_brightness, contrast, adaptive_on, manual_override`
  - `manual_override = true`는 사용자가 수동 슬라이더/단축키로 조정한 직후 1회 기록
- 주기: DDC 전송 시마다 1줄
- 용량 제어: 30일 이상 레코드 자동 삭제 또는 로테이션
- 용도: 나중에 곡선 튜닝 / 혹시 나중 학습 도입 시 데이터 소스

## 8. 기술 리스크 / 연구 과제

1. ~~**대비 Adaptive 곡선**~~ — **해소**. 고정 70으로 결정. (5.2.2)
2. **`corebrightnessdiag` 호출 비용** — 1초 주기로 돌릴 때 CPU/배터리 영향 측정.
3. **단축키 충돌** — `⌃⌥⌘+화살표`가 다른 앱과 겹치는지 확인.
4. **HID 직접 구독 (v2)** — `AppleSPUVD6286` HID Usage Page 문서 부재 → MonitorControl 소스 참고.
5. **Lunar와 DDC 충돌** — 동시 실행 시 최종 승자 = 마지막 명령 발신자. 온보딩에서 안내.
6. **m1ddc 의존성** — 향후 직접 구현 시 `IOAVService`/`IORegistry` 접근 필요.
7. **LG HDR 4K b%→nits 매핑이 정말 선형인가** — 5.2.1 표는 선형 가정. 실측 시 비선형이면 b_max 조정 필요. 1주일 사용 후 휘도계로 검증 권장.
8. **종이 반사율 0.85 가정** — 본인이 자주 쓰는 용지 등급에 따라 ±10% 변동. 큰 영향 없음.
